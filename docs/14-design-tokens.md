# AME Bazaar Design Tokens

## Purpose

This document defines the official design tokens for the AME Bazaar Digital Platform.

Design tokens are the single source of truth for colours, typography, spacing, sizing, borders, shadows, animations, breakpoints, and other visual properties.

Every page, component, and future feature must use these tokens instead of creating new values.

---

# Design Philosophy

The visual system should remain:

* Premium
* Minimal
* Elegant
* Clean
* Consistent
* Accessible
* Mobile First

Inspired by:

* Apple
* Zara
* Nike
* Uniqlo
* Westside

# Font Scale

Use a consistent type scale.

Levels include:

* Display
* H1
* H2
* H3
* H4
* H5
* H6
* Body Large
* Body
* Small Text
* Caption

Avoid arbitrary font sizes.

---

# Font Weight

Use a limited number of font weights.

Recommended:

* Regular
* Medium
* Semi Bold
* Bold

Avoid excessive variations.

---

# Line Height

Maintain comfortable reading.

Headings

Compact but readable.

Body Text

Generous spacing for long-form reading.

---

# Spacing Tokens

Examples:

* Extra Small
* Small
* Medium
* Large
* Extra Large
* Section Spacing

Never use random spacing values.

Maximum Content Width

Consistent across the platform.

Containers

Section Padding

# Grid Tokens

Tablet

Single-column layout.

# Border Radius

Use one consistent border-radius system.

Examples:

* Small
* Medium
* Large

Avoid mixing multiple radius styles.

---

# Border Tokens

Use subtle borders.

Maintain consistent thickness throughout the platform.

---

# Shadow Tokens

Use soft shadows only where they improve usability.

Levels:

* None
* Small
* Medium
* Large

Avoid heavy shadows.

---

# Elevation

Interactive components should use a consistent elevation system.

Examples:

* Cards
* Navigation
* Dropdowns
* Modals

---

# Button Tokens

Every button should use the same:

* Padding
* Border Radius
* Typography
* Hover Behaviour
* Focus State

Do not create unique button styles for different pages.

---

# Form Tokens

Maintain consistent:

* Input Height
* Padding
* Label Position
* Border Radius
* Error Messages
* Focus States

Forms should feel identical across the platform.

---

# Icon Tokens

Use one icon style only.

Maintain consistent:

* Size
* Stroke Weight
* Alignment

Do not mix icon libraries.

---

# Image Tokens

Images should maintain consistent:

* Aspect Ratio
* Border Radius
* Loading Behaviour
* Cropping Style

Prioritise authentic brand photography.

---

# Motion Tokens

Animations should be minimal.

Allowed:

* Fade
* Smooth Hover
* Gentle Slide
* Accordion

Avoid:

* Bounce
* Flash
* Excessive Motion

Motion should improve usability rather than distract.

---

# Transition Tokens

Use consistent transition timing throughout the website.

Hover effects, menus, buttons, and interactive elements should feel unified.

---

# Breakpoints

Support:

* Mobile
* Tablet
* Laptop
* Desktop
* Large Desktop

The design system should remain mobile-first.

---

# Z-Index System

Maintain a consistent stacking order.

Examples:

* Header
* Dropdown
* Modal
* Notification
* Overlay

Avoid arbitrary z-index values.

---

# Accessibility Tokens

Ensure:

* Proper colour contrast
* Readable typography
* Visible focus indicators
* Touch-friendly controls
* Keyboard accessibility

Accessibility is part of the design system, not an optional enhancement.

---

# Performance Guidelines

Design tokens should help reduce CSS duplication.

Prefer reusable variables instead of repeated values.

The visual system should remain lightweight and efficient.

---

# Future Expansion

The token system should support future additions including:

* WooCommerce
* Customer Accounts
* AI Shopping Assistant
* Franchise Portal
* Private Label Pages
* Multi-location Support

New features should extend the existing token system instead of introducing new visual rules.

---

# Documentation Rule

This document defines the official design tokens for the AME Bazaar Digital Platform.

All components, templates, pages, and future features must use these tokens to maintain visual consistency, accessibility, performance, and long-term scalability.

These tokens are the foundation of the platform's design language and should only be updated through an approved design system revision.
