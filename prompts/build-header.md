# Build Header Component

## Task

Build the complete Header component for the AME Bazaar Digital Platform.

Do not build any other section.

Build only the Header.

Stop after the Header is complete.

---

# Before You Start

Before writing any code:

1. Read every project document inside the `/docs` folder.
2. Read `prompts/master-system-prompt.md`.
3. Understand the AME Bazaar brand.
4. Understand the Design System.
5. Understand the Homepage architecture.
6. Understand the Component Library.
7. Understand the Design Tokens.
8. Review the Shopify reference files.
9. Review all available project assets.

Do not begin implementation until these files have been understood.

---

# Platform

Platform

WordPress

Theme

Astra Child Theme

Future Compatibility

WooCommerce

The implementation must remain compatible with future WooCommerce integration.

---

# Objective

The header should communicate:

* Premium brand identity
* Trust
* Simplicity
* Professionalism
* Modern retail experience

The design should feel closer to Apple, Zara, Uniqlo and Nike than to a traditional WordPress website.

---

# Design Requirements

The header must be:

* Minimal
* Elegant
* Responsive
* Accessible
* Fast
* Sticky on scroll
* Mobile First

Avoid:

* Template appearance
* Heavy shadows
* Excessive animations
* Visual clutter

---

# Branding

Always use the official AME Bazaar logo.

Never replace the logo.

Never generate placeholder branding.

If the logo is unavailable, stop and request the correct asset.

---

# Navigation

Desktop navigation should include:

* Home
* About
* Shop
* Categories
* Blog
* Contact

Support future menu expansion.

Navigation should be editable through WordPress.

---

# Mega Menu

Design the navigation so it can support future Mega Menus.

Do not hardcode Mega Menu behaviour.

The architecture should remain extensible.

---

# Search

Include a search component.

Current behaviour:

Website Search

Future behaviour:

WooCommerce Product Search

The architecture should support both.

---

# Header Actions

Prepare space for:

* Search
* WhatsApp
* Call
* Future Account
* Future Wishlist
* Future Cart

Do not fully implement future features.

Prepare the layout for them.

---

# Mobile Navigation

Create an accessible mobile menu.

Requirements:

* Off-canvas navigation
* Keyboard accessible
* Touch friendly
* Smooth performance

Avoid heavy JavaScript.

---

# Accessibility

Support:

* Keyboard navigation
* Screen readers
* Proper ARIA attributes
* Focus indicators
* Accessible menus

Accessibility is mandatory.

---

# Performance

Optimise for:

* Fast loading
* Minimal JavaScript
* Lightweight CSS
* Semantic HTML

Avoid unnecessary dependencies.

---

# SEO

Use semantic HTML.

Examples:

* header
* nav

Navigation should improve crawlability.

---

# AI Visibility

The header should reinforce:

* Brand identity
* Site hierarchy
* Internal navigation

The implementation should help both users and machines understand the website.

---

# Reusability

Build the header as reusable components.

Examples:

* Logo
* Navigation
* Search
* Mobile Menu
* Action Icons

Avoid tightly coupled implementations.

---

# Deliverables

Generate:

* PHP template(s)
* CSS
* JavaScript (only if required)
* WordPress integration
* Astra Child Theme compatible structure

Follow the folder structure defined in the project documentation.
