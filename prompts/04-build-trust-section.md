# Build Trust Section

## Task

Build the Trust Section for the AME Bazaar Homepage.

Do not build any other homepage section.

Build only the Trust Section.

Stop after completing this component.

---

Before generating any code:

1. Read all project documentation inside `/docs`.
2. Read `prompts/00-master-system-prompt.md`.
3. Review the Homepage Documentation.
4. Review the Design System.
5. Review the Component Library.
6. Review the Design Tokens.
7. Review the Shopify reference.
8. Review the official project assets.

Do not begin implementation until the project architecture has been understood.

# Objective

The Trust Section exists to reduce uncertainty immediately after the Hero.

Its purpose is to reassure visitors that AME Bazaar is a reliable and established family fashion retailer.

The section should build confidence before visitors continue exploring the homepage.

---

# Design Direction

The Trust Section should feel:

* Premium
* Minimal
* Spacious
* Elegant
* Clean
* Modern

Inspired by:

* Apple
* Zara
* Nike
* Uniqlo

Avoid:

* Large coloured boxes
* Loud icons
* Flashy badges
* Discount banners
* Marketing gimmicks
* Generic ecommerce trust bars

---

# Content Rules

Use only verified business information.

Do not invent:

* Customer numbers
* Awards
* Years in business
* Certifications
* Ratings
* Guarantees

If official information is unavailable, leave the component prepared for future dynamic content.

---

# Component Structure

The Trust Section should support reusable trust cards.

Each card should include:

* Icon
* Short heading
* Supporting description

The layout must remain modular.

---

# Suggested Trust Topics

Use only if supported by project documentation.

Examples:

* Premium Quality
* Fair Pricing
* Family Fashion
* Personalised Customer Service
* Tailoring Support
* Trusted Local Retailer

The wording should remain factual and customer-focused.

---

# Visual Style

The Trust Section should:

* Use generous whitespace
* Maintain consistent spacing
* Follow Design Tokens
* Use subtle iconography
* Avoid heavy borders
* Avoid excessive shadows

The section should complement the Hero rather than compete with it.

---

# Responsive Design

Support:

* Mobile
* Tablet
* Laptop
* Desktop

Cards should stack naturally on smaller screens.

---

# Accessibility

Ensure:

* Semantic HTML
* Keyboard accessibility
* Descriptive icon labels where required
* Proper heading hierarchy
* Sufficient colour contrast

Accessibility is mandatory.

---

# Performance

Prioritise:

* Lightweight markup
* Optimised SVG icons
* Minimal CSS
* No unnecessary JavaScript

The Trust Section should load instantly.

---

# SEO

The Trust Section should reinforce:

* Business credibility
* Product quality
* Customer service
* Local retail identity

Use semantic headings and readable text.

---

# AI Visibility

The Trust Section should help AI systems understand:

* Why customers trust AME Bazaar
* What differentiates the business
* What services are offered
* What values define the brand

Content should remain factual and easy to interpret.

---

# Reusability

The Trust Section should be built as reusable components.

Each trust card should function independently.

Future cards should be addable without redesigning the section.
