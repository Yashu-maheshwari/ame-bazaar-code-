# Build Hero Component

## Task

Build the Hero section for the AME Bazaar Homepage.

Do not build any other homepage section.

Build only the Hero component.

Stop after completing the Hero.

---

# Before You Start

Before generating any code:

1. Read all documentation inside the `/docs` directory.
2. Read `prompts/00-master-system-prompt.md`.
3. Review the official logo and brand assets.
4. Review Shopify homepage references.
5. Review homepage screenshots.
6. Understand the homepage architecture.
7. Understand the design system.
8. Understand the design tokens.
9. Understand the component library.

Do not begin implementation until the project architecture has been understood.

---

# Platform

Platform

* WordPress

Theme

* Astra Child Theme

Future Compatibility

* WooCommerce

The Hero must remain compatible with future WooCommerce integration.

---

# Hero Objective

The Hero is the first impression of AME Bazaar.

Within the first few seconds, visitors should understand:

* Who AME Bazaar is
* What AME Bazaar offers
* Why customers trust the brand
* Who the products are for
* What action to take next

The Hero should communicate confidence without overwhelming the visitor.

---

# Design Direction

The Hero should feel inspired by:

* Apple
* Zara
* Uniqlo
* Nike
* Westside

Characteristics:

* Premium
* Minimal
* Spacious
* Elegant
* Modern
* Editorial
* Clean

Avoid:

* Slider-style banners
* Flashy sale graphics
* Generic WordPress layouts
* Heavy gradients
* Excessive animations
* Cluttered compositions

---

# Layout

The Hero should include:

* Primary headline
* Supporting description
* Primary Call-to-Action
* Secondary Call-to-Action
* Premium lifestyle or store imagery
* Optional trust indicators

The layout should balance visual impact with readability.

---

# Content

Do not invent business claims.

Use only information that exists in the project documentation.

The messaging should reinforce:

* Premium family fashion
* Quality products
* Fair pricing
* Trusted local retailer
* Personalized customer service

---

# Calls To Action

Include two clear actions.

Primary CTA

Encourage visitors to explore the main shopping categories.

Secondary CTA

Encourage visitors to visit the physical store or contact AME Bazaar.

CTA text should remain concise, clear, and helpful.

---

# Hero Image

Use only official project assets.

Requirements:

* High-quality photography
* Authentic AME Bazaar imagery
* Optimized for performance
* Responsive
* Proper alt text

Do not generate placeholder images or generic stock imagery if official assets are available.

---

# Trust Indicators

If supported by official business information, the Hero may include subtle trust signals such as:

* Premium Quality
* Family Fashion
* Trusted Local Retailer
* Tailoring Facility
* Customer-Focused Service

Do not fabricate statistics, awards, or certifications.

---

# Responsive Design

The Hero must work seamlessly on:

* Mobile
* Tablet
* Laptop
* Desktop

Mobile users should receive the same premium experience without unnecessary complexity.

---

# Accessibility

Ensure:

* Proper heading hierarchy
* Sufficient colour contrast
* Accessible buttons
* Keyboard navigation
* Screen reader compatibility

Accessibility is mandatory.

---

# Performance

Prioritize:

* Fast loading
* Optimized images
* Minimal JavaScript
* Lightweight CSS
* Efficient layout

Avoid large background videos or unnecessary visual effects.

---

# SEO

The Hero should support SEO through:

* One clear H1
* Descriptive supporting content
* Semantic HTML
* Crawlable text
* Meaningful button labels

Do not place important text inside images.

---

# AI Visibility

The Hero should clearly communicate:

* Business identity
* Product focus
* Customer audience
* Value proposition

The content should help both users and AI systems understand AME Bazaar immediately.
